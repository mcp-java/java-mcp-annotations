Framework-agnostic server APIs for building MCP servers.

This package provides runtime interfaces and utilities that can be used by MCP server implementations. These APIs are designed to be framework-independent and can be implemented by any Java-based MCP server framework.

Core APIs

  • {@link org.mcp_java.server.Cancellation} - Request cancellation handling
  • {@link org.mcp_java.server.ContentEncoder} - Custom content encoding
  • {@link org.mcp_java.server.McpException} - Base MCP exception
  • {@link org.mcp_java.server.McpConnection} - Connection metadata and status, including client capabilities
  • {@link org.mcp_java.server.McpLog} - Client-side logging
  • {@link org.mcp_java.server.progress.Progress} - Progress reporting for long-running operations

Access to client features

  • {@link org.mcp_java.server.sampling.Sampling} - Request LLM sampling from client
  • {@link org.mcp_java.server.elicitation.Elicitation} - Request additional information from user
  • {@link org.mcp_java.server.roots.Roots} - Access to client's root directories

Responses for server features

These APIs are intended for use with the corresponding annotations.

  • {@link org.mcp_java.server.prompts.Prompts} - Build responses to prompt requests
  • {@link org.mcp_java.server.resources.Resources} - Build responses to resource requests
  • {@link org.mcp_java.server.tools.Tools} - Build responses to tool calls
  • {@link org.mcp_java.server.content.Contents} - Build content blocks for inclusion in resources and tool call responses
  • {@link org.mcp_java.server.completion.Completions} - Build responses to completion requests

Framework-Agnostic Design

These interfaces use generic types for reactive returns (e.g., {@code T send()}). Framework implementations should return their native reactive types:

  • Quarkus: {@code Uni} from SmallRye Mutiny
  • Spring: {@code Mono} from Project Reactor
  • Standard Java: {@code CompletableFuture}
